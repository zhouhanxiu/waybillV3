## 二、鉴权方式

### 2.1 API Key 鉴权

所有 V2→V3 和 V3→V2 的内部接口统一使用 Bearer Token 鉴权。

**请求头格式**：
```
Authorization: Bearer v3-internal-key
```

**Token 配置**：
- 环境变量：`INTERNAL_API_KEY`（默认值 `v3-internal-key`）
- V2 和 V3 使用相同的密钥

**未授权响应**：
```json
{ "error": "未授权" }
```
HTTP 状态码：`401 Unauthorized`

## 三、V2 提供的接口（V3 调用）

### 3.1 运单数据同步

拉取 V2 的全部运单数据（含 SKU 明细），供 V3 写入本地快照。

| 项目 | 内容 |
|------|------|
| **端点** | `POST /api/waybills/sync` |
| **鉴权** | Bearer Token |
| **超时** | V3 侧设置 8 秒 |

**请求体**（可选，为空则返回全部）：
```json
{
  "external_codes": ["DP20260701001", "DP20260701002"]
}
```

**成功响应 (200)**：
```json
[
  {
    "id": "uuid-xxxx",
    "external_code": "DP20260705001",
    "store_name": "朝阳旗舰店",
    "receiver_name": "张三",
    "receiver_phone": "13800138001",
    "receiver_address": "北京市朝阳区XX路XX号",
    "amount": 20,
    "created_at": "2026-07-05T10:00:00Z",
    "items": [
      {
        "id": "item-uuid-xxxx",
        "waybill_id": "uuid-xxxx",
        "sku_code": "SKU001",
        "sku_name": "东北大米",
        "quantity": 20,
        "spec": "5kg"
      }
    ]
  }
]
```

**调用示例（V3 端）**：
```typescript
const res = await fetch(`${V2_URL}/api/waybills/sync`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${INTERNAL_KEY}`,
  },
  body: JSON.stringify({}),
});
```

### 3.2 运单异常状态回写

V3 审批工单后，将运单的异常状态回写到 V2。

| 项目 | 内容 |
|------|------|
| **端点** | `POST /api/waybills/exception-status` |
| **鉴权** | Bearer Token |

**写入请求体**：
```json
{
  "external_code": "DP20260705001",
  "has_open_ticket": true,
  "ticket_count": 3
}
```

**查询请求**（GET）：
```
GET /api/waybills/exception-status?external_code=DP20260705001
```
（鉴权同样需要 Bearer Token）

**查询响应 (200)**：
```json
{
  "external_code": "DP20260705001",
  "has_open_ticket": true
}
```

**调用时机**：V3 的考点4数据一致性测试中，批量创建工单后调用。

### 3.3 SKU 校验

验证指定 SKU 编码是否属于某条运单。

| 项目 | 内容 |
|------|------|
| **端点** | `GET /api/waybills/verify-sku?external_code={code}&sku_code={sku}` |
| **鉴权** | Bearer Token |

**成功响应 (200)**：
```json
{
  "valid": true,
  "waybill_id": "uuid-xxxx",
  "item": { "sku_code": "SKU001", "sku_name": "东北大米", "quantity": 20 }
}
```

**无效响应 (200)**：
```json
{
  "valid": false,
  "reason": "SKU 不属于该运单"
}
```

### 4.2 运单快照 API

用于 V3 本地缓存 V2 运单数据，实现 V2 不可用时的降级。

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/waybills/snapshot` | 读取本地快照 |
| `POST` | `/api/waybills/snapshot` | 写入/刷新快照 |
| `DELETE` | `/api/waybills/snapshot` | 清除快照 |

**写入请求体**：
```json
{
  "waybills": [
    {
      "id": "wb_001",
      "external_code": "DP20260705001",
      "store_name": "朝阳旗舰店",
      "receiver_name": "张三",
      "items": [
        { "id": "item_001", "sku_code": "SKU001", "sku_name": "东北大米", "quantity": 20 }
      ]
    }
  ]
}
```

**写入响应 (200)**：
```json
{
  "ok": true,
  "upserted": 1,
  "items": 1,
  "errors": 0,
  "message": "快照已更新：1 条运单，1 条商品"
}
```

**读取响应 (200)**：返回与写入结构一致的运单数组（含 items 子数组）。

## 五、数据同步流程

### 5.1 V3 启动时的数据同步

```
1. V3 调用 POST /api/waybills/sync (V2)
2. V2 返回运单列表（含 SKU 明细）
3. V3 调用 POST /api/waybills/snapshot 写入本地快照
4. V3 基于快照数据创建工单
```

### 5.2 V2 故障时的降级流程

```
1. V3 调用 POST /api/waybills/sync (V2) → 超时/连接拒绝
2. V3 调用 GET /api/waybills/snapshot → 读取本地快照
3. 快照有数据 → 使用快照数据继续
4. 快照无数据 → 使用硬编码 fallback 数据（3条测试运单）
```

### 5.3 工单闭环后的回写

```
1. V3 工单审批通过 → 状态变为 approved
2. V3 调用 POST /api/waybills/exception-status 通知 V2
3. V2 标记该运单存在异常工单
```

## 六、错误处理约定

| 场景 | HTTP状态码 | 响应体 |
|------|-----------|--------|
| 未授权 | 401 | `{ "error": "未授权" }` |
| 缺少必要参数 | 400 | `{ "error": "缺少工单必要参数" }` |
| 工单不存在 | 404 | `{ "error": "工单不存在", "status": 404 }` |
| 自审限制 | 403 | `{ "error": "上报人不能审批自己的工单", "status": 403 }` |
| 并发冲突 | 409 | `{ "error": "工单已完结，不可重复审批", "status": 409 }` |
| 无放行权限 | 403 | `{ "error": "无放行权限，需要主管或管理员", "status": 403 }` |
| V2 不可达 | N/A | V3 自动降级到快照/fallback |
| 快照写入失败 | 500 | `{ "error": "错误详情" }` |
