# 大模型调用说明

| 配置项 | 说明 |
|--------|------|
| 模型 | DeepSeek Chat（`deepseek-chat`），也支持 OpenAI 兼容 API |

## 二、AI 介入环节

### 2.1 异常类型智能推荐（扫描品控）

**触发时机**：操作人填写扫描描述文本后，AI 根据描述推荐品控异常子类型和严重度。

**Prompt 设计思路**：
```
你是一位仓库品控专家。请根据以下扫描描述，推荐品控异常子类型和严重度。

品控异常子类型：qty_mismatch(数量不符)、appearance(外观破损)、
spec_mismatch(规格不符)、label_error(标签错误)、batch_error(批次异常)

严重度：low(低)、medium(中)、high(高)、critical(严重)

扫描描述：{description}

请返回 JSON 格式：{ exception_subtype, severity, confidence, reason }
```

**AI 建议展示规则**：
- 前端明确标注"🤖 AI 建议，需人工确认"
- 操作人可以修改 AI 推荐的类型和严重度
- AI 服务超时或调用失败时，不阻塞扫描流程，操作人手动选择

### 2.2 审批建议（审批环节）

**触发时机**：审批人打开工单详情时，AI 根据历史审批记录给出建议审批意见。

**Prompt 设计思路**：
```
你是一位物流审批顾问。请根据以下工单信息和历史审批记录，给出建议审批意见。

工单信息：
- 异常类型：{exception_type}
- 涉及金额：{amount}
- 异常描述：{description}

历史审批记录（最近 5 条相似工单）：
{history}

请返回 JSON 格式：{ suggestion: "approve"|"reject", reason, references }
其中 references 需列出参考了哪几条历史记录（记录 ID）。
```

**AI 建议展示规则**：
- 前端明确标注"🤖 AI 建议，需人工确认"
- 必须展示 AI 的推理依据（引用了哪些历史记录）
- 审批人可以选择采纳或忽略 AI 建议
- AI 服务不可用时，审批人直接手动操作

## 三、AI 调用安全原则

| 原则 | 实现方式 |
|------|----------|
| AI 建议仅供人工参考 | 所有 AI 输出都标注"AI 建议，需人工确认" |
| AI 不能自动执行 | 审批、放行等操作必须人工点击确认 |
| AI 超时不能阻塞主流程 | 所有 AI 调用都有 5 秒超时，超时后跳过 AI 建议 |
| AI 建议可追溯 | 审批记录表的 `ai_suggestion` 字段保存 AI 建议原文 |
| AI 建议不能是黑箱结论 | 必须说明推理依据（引用了哪些历史记录） |
