# 异常工单系统 V3 — 大模型调用说明

## 1. 概述

V3 异常工单系统在两个业务环节中引入大模型辅助决策：

| 环节 | AI 作用 | 目的 |
|------|---------|------|
| **扫描品控** | 根据操作人输入的异常描述，智能推荐异常子类型和严重度 | 减少人工判断失误，加快品控录入 |
| **审批** | 根据工单信息和历史审批记录，给出建议审批意见 | 辅助审批人做出一致性决策 |

> **核心原则：AI 是辅助工具，所有 AI 建议仅供人工参考，最终决策由操作人/审批人来执行。**

## 4. AI 介入环节详解

### 4.1 异常类型智能推荐（扫描品控环节）

#### 触发时机
操作人在扫描品控页面输入异常描述文本后，系统自动（或手动触发）调用 AI 推荐异常子类型和严重度。

#### 业务流程

```
操作人扫描运单 → 发现异常 → 填写描述文本
    ↓
触发 AI 推荐（自动/手动）
    ↓
AI 返回：exception_subtype + severity + confidence + reason
    ↓
前端展示推荐结果（标注"AI 建议"）
    ↓
操作人确认或修改 → 提交扫描记录
```

#### Prompt 设计

```text
你是一位仓库品控专家。请根据以下扫描描述，推荐品控异常子类型和严重度。

品控异常子类型（必须从以下选择）：
- qty_mismatch: 数量不符（实际数量与预期不一致）
- appearance:   外观破损（外包装破损、货物损伤）
- spec_mismatch: 规格不符（型号、尺寸、重量等与订单不一致）
- label_error:  标签错误（标签信息错误、缺失、无法识别）
- batch_error:  批次异常（批次号不匹配、过期、混批）

严重度判定规则：
- low:      轻微问题，不影响使用（如标签轻微污损）
- medium:   一般问题，需要关注（如数量少1~2件）
- high:     严重问题，影响交付（如外观明显破损）
- critical: 紧急问题，必须立即处理（如货物完全损毁、批量错误）

扫描描述：{description}

请返回 JSON 格式：
{
  "exception_subtype": "qty_mismatch",
  "severity": "medium",
  "confidence": 0.85,
  "reason": "描述中提到'少了3件'，属于数量差异，差异数较小，建议中等严重度"
}
```

#### 前端展示规则

| 规则 | 说明 |
|------|------|
| 明确标注 | 推荐结果前显示"🤖 AI 建议，需人工确认" |
| 可编辑 | 操作人可以修改 AI 推荐的类型和严重度 |
| 置信度提示 | `confidence < 0.6` 时黄色警告"AI 建议可信度较低，请仔细核对" |
| 推理展示 | 展示 AI 给出的 `reason` 字段 |

#### 容错策略

```
AI 调用超时（5 秒）
    ↓
跳过 AI 推荐，不阻塞扫描流程
    ↓
操作人手动选择异常类型和严重度
```

### 4.2 审批建议（审批环节）

#### 触发时机
审批人打开工单详情页时，系统根据工单信息和历史审批记录，调用 AI 给出建议审批意见。

#### 业务流程

```
审批人打开工单 → 系统加载工单信息 + 最近 5 条相似工单历史
    ↓
调用 AI 审批建议
    ↓
AI 返回：suggestion + reason + references
    ↓
前端展示（标注"AI 建议"），审批人可采纳或忽略
    ↓
审批人执行审批操作（approve/reject）
    ↓
审批记录保存，含 ai_suggestion 字段（可追溯）
```

#### Prompt 设计

```text
你是一位物流审批顾问。请根据以下工单信息和历史审批记录，给出建议审批意见。

工单信息：
- 异常类型：{exception_type}
- 严重度：{severity}
- 涉及金额：{amount}
- 异常描述：{description}

历史审批记录（最近 5 条相似工单）：
{history}

审批建议规则：
- 金额 <= 200 且历史多数 approve → 建议通过
- 金额 > 1000 且属于严重问题 → 建议驳回或升级
- 异常类型在历史中高频 reject → 建议驳回
- 无足够历史参考 → 建议人工判断（标记 confidence 为 low）

请返回 JSON 格式：
{
  "suggestion": "approve",
  "reason": "同类异常历史审批通过率 80%，金额在常规范围内",
  "confidence": 0.75,
  "references": [
    { "ticket_id": "ticket_xxx", "action": "approve", "similarity": "高" },
    { "ticket_id": "ticket_yyy", "action": "reject", "similarity": "中" }
  ]
}
```

#### 数据来源

查询相似工单的 SQL 逻辑（就近匹配异常类型和严重度）：

```sql
SELECT ar.action, ar.opinion, t.exception_type, t.severity, t.amount
FROM approval_records ar
JOIN exception_tickets t ON t.id = ar.ticket_id
WHERE t.exception_type = $1
ORDER BY t.created_at DESC
LIMIT 5
```

#### 前端展示规则

| 规则 | 说明 |
|------|------|
| 明确标注 | 建议前显示"🤖 AI 建议，需人工确认" |
| 推理依据 | 展示 AI 引用了哪些历史记录（含相似度） |
| 采纳/忽略 | 审批人点击"采纳建议"或手动选择不同操作 |
| 非阻塞 | AI 服务不可用时，审批人直接手动操作 |

#### 数据追溯

审批记录表中的 `ai_suggestion TEXT` 字段存储 AI 建议的原始 JSON：

```json
{
  "suggestion": "approve",
  "reason": "同类异常历史审批通过率 80%...",
  "confidence": 0.75,
  "references": [...]
}
```

## 5. 调用参数设计

```typescript
import { generateObject } from "ai";
import { createOpenAI } from "@ai-sdk/openai";
import { z } from "zod";

// 异常类型推荐 Schema
const QcRecommendSchema = z.object({
  exception_subtype: z.enum([
    "qty_mismatch", "appearance", "spec_mismatch",
    "label_error", "batch_error"
  ]),
  severity: z.enum(["low", "medium", "high", "critical"]),
  confidence: z.number().min(0).max(1),
  reason: z.string(),
});

// 审批建议 Schema
const ApprovalTipSchema = z.object({
  suggestion: z.enum(["approve", "reject"]),
  reason: z.string(),
  confidence: z.number().min(0).max(1),
  references: z.array(z.object({
    ticket_id: z.string(),
    action: z.enum(["approve", "reject"]),
    similarity: z.enum(["高", "中", "低"]),
  })),
});

// 调用参数
const { object } = await generateObject({
  model: openai("deepseek-chat"),  // 或 glm-4.5-air
  schema: QcRecommendSchema,       // Zod Schema
  prompt,
  temperature: 0.1,    // 极低温度，追求确定性和一致性
  maxRetries: 1,        // 失败重试 1 次
  maxTokens: 500,       // 限制输出长度，控制延迟
});
```

| 参数 | 值 | 原因 |
|------|-----|------|
| `temperature` | 0.1 | 分类/推荐任务需要高确定性，低温度减少幻觉 |
| `maxRetries` | 1 | 失败重试一次，避免无谓等待 |
| `maxTokens` | 500 | 输出简洁 JSON，控制延迟在 2~3 秒内 |

## 6. 安全与容错

### 6.1 五条核心原则

| # | 原则 | 实现方式 |
|---|------|----------|
| 1 | **AI 建议仅供人工参考** | 所有 AI 输出均标注"🤖 AI 建议，需人工确认" |
| 2 | **AI 不能自动执行** | 审批、放行等操作必须人工点击确认，不存在 `auto_approve` 路径 |
| 3 | **AI 超时不能阻塞主流程** | 所有 AI 调用设置 **5 秒超时**，超时后跳过 AI，操作人手动完成 |
| 4 | **AI 建议可追溯** | `approval_records.ai_suggestion` 字段保存 AI 建议原文，用于审计 |
| 5 | **AI 非黑箱** | 必须展示推理依据（`reason`）和引用记录（`references`） |

### 6.2 容错策略

```
┌─ 调用 AI ──────────────────────────────────────────┐
│                                                      │
│  ┌──────────┐     ┌──────────┐     ┌─────────────┐  │
│  │ 超时 5s   │ →→  │ 重试 1 次 │ →→  │ 降级：手动   │  │
│  └──────────┘     └──────────┘     └─────────────┘  │
│                                                      │
│  不阻塞主流程，用户无感知或仅看到"AI建议暂不可用"      │
└──────────────────────────────────────────────────────┘
```

### 6.3 降级体验

| 场景 | 前端表现 |
|------|----------|
| AI 正常返回 | 展示推荐结果，标注"AI 建议" |
| AI 超时 | 静默跳过，AI 推荐区域显示"AI 建议暂不可用，请手动选择" |
| AI 返回格式异常 | Schema 校验失败 → SDK 自动重试 → 仍失败则降级 |
| API Key 未配置 | 不发起调用，AI 推荐区域不显示（环境优雅降级） |

## 7. SDK 初始化

```typescript
// src/lib/ai.ts（待创建）

import { generateObject } from "ai";
import { createOpenAI } from "@ai-sdk/openai";

function getAIClient() {
  // 环境变量未配置时返回 null，前端据此隐藏 AI 功能
  const apiKey = process.env.DEEPSEEK_API_KEY || process.env.OPENAI_API_KEY;
  if (!apiKey) return null;

  const baseURL = process.env.AI_BASE_URL || "https://api.deepseek.com/v1";
  const modelName = process.env.AI_MODEL || "deepseek-chat";

  const openai = createOpenAI({ apiKey, baseURL });
  return { generateObject, model: openai(modelName) };
}

// 带超时的安全调用
export async function safeGenerateObject<T>(
  schema: z.ZodSchema<T>,
  prompt: string,
): Promise<T | null> {
  const client = getAIClient();
  if (!client) return null; // 未配置 AI，静默跳过

  try {
    const { object } = await Promise.race([
      client.generateObject({
        model: client.model,
        schema,
        prompt,
        temperature: 0.1,
        maxRetries: 1,
        maxTokens: 500,
      }),
      new Promise<never>((_, reject) =>
        setTimeout(() => reject(new Error("AI timeout")), 5000)
      ),
    ]);
    return object;
  } catch (err) {
    console.warn("AI 调用失败，降级到手动模式:", (err as Error).message);
    return null;
  }
}
```
